JavaScript Variables

JavaScript variables' scope defines where they are accessible and where they are not. A variable's scope can be divided into three categories: global scope, function scope, and block scope. Depending on whether you use var, let, or const, a variable behaves differently.

var :

  • 1. Variables declared with var have function scope or global scope but not block scope. This indicates that they are usable inside the scope of the stated function or the entirety of the global context.
  • 2. When a var variable is declared inside a function, it cannot be accessed from outside that function.
  • 3. Anywhere in the script can access a var variable if it is declared outside of any function.
  • 4. Var variables can be used before they are declared since they are hoisted to the top of their scope (but their value won't be known until it is assigned).

let :

  • 1. Let variables have block scope. They are only available within the declaration block.
  • 2. Let variables, unlike var, are not hoisted to the top of their scope, therefore they cannot be used before they are defined.
  • 3. You will get a syntax error if you define a let variable with the same name in the same scope.

const :

  • 1. Like let, variables declared with const also have block scope.
  • 2. The main distinction is that once const variables are declared, they must have a value assigned to them since once that value is assigned, it cannot be changed. However, properties or elements for objects and arrays can still be changed.
  • 3. Similar to let, const variables are only usable within the block in which they are declared and are not elevated to the top of their scope.
